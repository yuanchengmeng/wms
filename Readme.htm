Sample Overview

This sample provides an implementation of scanning interface that allows users to scan barcode. It also can set/get symbology, preamble and postamble with barcode reader. It is built with Microsoft Visual Studio 2008 and Microsoft .NET Compact Framework 3.5.

 

Requirements and Dependencies

The itcscan.dll and bcsymbologymapping.xml must be installed on the devices.  The updated bcsymbologymapping.xml file can be found in the runtime folder of C:\Program Files\Intermec\Developer Library\Data Collection Resource Kit\Runtime directory. The Intermec.DataCollection2.CF3.5.dll assembly file must in the application directory. 

 

Microsoft Visual Studio 2008 must be installed on your development PC before you run the BarcodeReader_ce application. Microsoft .NET Compact Framework 3.5 is installed on the device.

 

Topics

BarcodeReader(string ScannerName), BarcodeReadEventHandler, ThreadedRead, strDatabuffer, Code39.Enable, Code128.Enable, Interleaved2of5.Enable, Pdf417.Enable, Preamble, Postamble

 

 

Building the Sample

The project can be built in the standard manner: Load the project solution, BarcodeReader.sln, in Visual Studio .NET 2008. Make sure a reference to Intermec.DataCollection2.CF3.5.dll has been added. Select the desired platform and then select “build”.

 

 

Running the Sample

Download the application to the Intermec portable device; all of the application files will be correctly installed if you have an ActiveSync connection to the device and you elect to Deploy the application from Visual Studio .NET 2008. This application has a single form that allows the user to interact with the BarcodeReader Class.

 

Notes

The latest bcsymbologymapping.xml file is provided in runtime folder of C:\Program Files\Intermec\Developer Library\Data Collection Resource Kit\Runtime directory, which has more options of symbology.
